Pattern Reference

All HexIntent patterns in one place: start direction, stack contract, return value, and a direct scribe link. Signatures are taken from server registrations and use the north-east start direction.

CREATE_LIST_MENU

{ne,awwaqwedwwd}
Menu constructor

Builds and opens a vertical list menu for the caster. It consumes a title and a list of typed UI entries from the stack and sends the resulting menu to the client; if no player caster exists, it mishaps.

Takes
top -> title
       [entry, entry, ...]
Returns

Pushes nothing. The consumed arguments are removed from the stack.

Notes

Only typed UI iotas are accepted inside the entry list. Invalid individual entries are skipped; a bad outer list mishaps. Missing player caster context also mishaps with "Manifestation requires a caster's will."

CREATE_GRID_MENU

{ne,awwaeawwaqwddad}
Menu constructor

Builds and opens a grid-layout menu for the caster. It works like CREATE_LIST_MENU, but also reads a column count and clamps it into the valid UI range; if no player caster exists, it mishaps.

Takes
top -> columns
       title
       [entry, entry, ...]
Returns

Pushes nothing. Column count, title, and entry list are consumed in that order from the stack.

Notes

columns must be a number iota. Value is rounded and clamped to 1..10 before build. Missing player caster context mishaps with "Manifestation requires a caster's will."

UI_BUTTON

{ne,awwaqwedwwdaa}
Intent constructor

Builds a typed button entry for a menu list. Clicking it dispatches stored action iotas in order.

Takes
top -> label
       [action, action, ...]
Returns

Pushes an IntentButton(label, actions) onto the stack.

Notes

Action list must be a non-empty list iota. Label can be any iota; display() text is used in UI.

UI_INPUT

{ne,awwaqwedwwdad}
Intent constructor

Builds a typed text input entry. The menu renders an input field with the label as prompt.

Takes
top -> label
Returns

Pushes an IntentInput(label) onto the stack.

Notes

Label is display-only. This constructor does not enforce a specific string type.

UI_SLIDER

{ne,awwaqwedwwdaw}
Intent constructor

Builds a numeric slider entry with explicit bounds and a current value. The rendered menu exposes a bounded numeric control to the player.

Takes
top -> label
       current
       max
       min
Returns

Pushes an IntentSlider(label, min, max, current) onto the stack.

Notes

min, max, and current must all be number iotas. min cannot exceed max, and current must lie within the range.

Takes

Returns

Notes

UI_DROPDOWN

{ne,awwaqwedwwdawaq}
Intent constructor

Builds a dropdown selector entry with a fixed list of options. The constructor stores the options and initializes the selected index to the first item.

Takes
top -> label
       [option, option, ...]
Returns

Pushes an IntentDropdown(label, options, 0) onto the stack.

Notes

The options slot must be a non-empty list iota. Each option is displayed using its display() text, and the initial selection is index 0.
